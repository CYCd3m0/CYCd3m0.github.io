Hugo with LoveIt

為了避免自己忘記，先記錄下關於建立這個 blog 的過程

Basic

  • 建立一個空的資料夾 (下面以 hugo-blog 為例)

  • 建立網站 : hugo new site <dir_name> 

    1

    hugo new site hugo-blog

  • 進到資料夾內，把要用的主題 clone 下來 (LoveIt) 

    1

    git clone https://github.com/dillonzq/LoveIt.git themes/LoveIt

  • 建立新的文章 : hugo new posts/<file_name> (以 Hugo-on-GitHub.md 為例)
    先完成下面的 Configuration 再新增文章 

    1

    hugo new posts/Hugo-on-GitHub.md

Avatar

  • 大頭貼的路徑已經在 config.toml 設定好了

  • 把圖片放在 static/images 資料夾中，檔名為 avatar.png 就可更改大頭貼

Favicons

  • 利用 Favicon Generator 生成 icon

  • 下載檔案

  • 解壓縮後，把資料夾內所有檔案丟到 static 資料夾中即可

Algolia

  • 設定都在 config.toml 完成了

  • 要修改的部分是 [params.search.algolia]

    index = "xxxxxblog"       # 自行建立並命名index
appID = "xxxxxxxDGP"      # 在API keys可以看到自己的appID跟searchKey
searchKey = "xxxxxxxxxxxxxxxxxxxxxxxxxxxx49e2"

  • index.json (records) 的部分下面再提

Testing

  • 在 localhost:1313 可查看目前成果 
    1

    hugo server --disableFastRender

Upload

  • 網站完成後，在根目錄 (hugo-blog) 生成 public 資料夾 

    1

    hugo

  • 在 GitHub 建立 <user_name>.github.io 的專案

  • 把該專案 clone 下來，再把 public 資料夾中的所有內容複製進去 (我把專案放在網站根目錄) 

    1

    cp -r public/* <user_name>.github.io

  • 到專案的資料夾中並把他 push 上去 

    1
2
3

    git add .
git commit -m "commit"
git push

  • 最後在該專案上可以看到一個 index.json ，把他 upload 到 algolia

Update

後續更新網站，要記得把 index.json 上傳，否則 Search 功能不會更新
